# Tipsheet Recommender

This project uses the [SentenceTransformers](https://sbert.net/) module to create embeddings from tipsheet data and build a recommendation system based on semantic textual similarity. 

## Connection.py

This file connects to the `ai4reporters` database using credentials and retrieves tipsheet titles, IDs, and website JSON data. The JSON data is then parsed to extract relevant information and written to `data_clean.txt`.

## CandidateGeneration.py

This file uses the data generated by `Connection.py` to display five similar tipsheets for a selected tipsheet.

First, the program filters candidate tipsheets by comparing their tags using Jaccard similarity. Only tipsheets with a tag similarity of at least 25% are considered. Titles are also parsed to remove commonly repeated phrases, which helps produce more accurate title similarity scores. Embeddings are then created from each tipsheet's title and summary analysis. The title and summary similarity scores are combined into an overall similarity score. Finally, the top five highest-scoring tipsheets are displayed as the best recommendations.

> [!NOTE]
> `CandidateGeneration.py` stores the title and summary embeddings locally in files called `title_emb.txt` and `sum_emb.txt`, respectively. When new tipsheets are added, only the new embeddings are appended to these files, while the existing embeddings are reused. This avoids recomputing embeddings for previously processed tipsheets and allows recommendations to be generated faster.
